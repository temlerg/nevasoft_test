Приглашаем Вас сделать тестовое задание в компании Невасофт.

Необходимо сделать макет системы авторизации, состоящий из нескольких экранов. Обращаю внимание – это должен быть макет. Никаких данных никуда отправляться не должно, вы работаете с бутафорскими данными, которые будут описаны ниже.
N.B. Это задание не содержит никаких описаний кроме списка необходимых экранов и сценариев работы с ними. Внешний вид, анимации, переходы, способы отображения полей для ввода, все необходимые элементы вы располагаете самостоятельно исходя из той задачи, которая должна быть реализована на каждом из экранов.

Нужно реализовать 4 экрана:
1) Экран ввод номера телефона пользователем, для запроса получения кода авторизации через смс
2) Ввод кода авторизации, который пользователь должен получить себе на указанный номер телефона для входа в свой аккаунт
3) Экран с текстом ошибки.
4) Экран-заглушка с сообщением что авторизация прошла успешно

И эмулировать в коде следующие сценарии при соответствующем пользовательском вводе:
1) При вводе номера телефона 79000000000 нужно выдавать ошибку с сообщением, что такого пользователя нет (точный текст ошибки здесь и далее на ваше усмотрение – это в общем то неважно)
2) При вводе номера телефона 79111111111 считать, что код отправлен и показывать экран, где можно ввести код из смс
3) При вводе кода 000000 показывать ошибку, что введенные код неверный
4) При вводе кода 111111 считать, что код введен корректно и показать экран-заглушку с сообщением что авторизация прошла успешно
